# Database Migrations

This folder contains versioned SQL migrations generated by Drizzle.
**Commit all files in this folder** — they are the source of truth for
the database schema across environments.

## Workflow

Whenever you change `lib/db/schema.ts`:

```bash
# 1. Generate a migration file from the schema diff
bun run db:generate

# 2. Review the generated .sql under lib/db/migrations/
#    Make sure the SQL matches your intent (especially for renames,
#    drops, or anything destructive). Edit by hand if needed.

# 3. Commit both schema.ts and the new migration file together

# 4. Apply on each environment
bun run db:migrate
```

## One-time bootstrap (existing databases)

If a database was previously managed with `drizzle-kit push` (so its
schema already matches but it has no `drizzle.__drizzle_migrations`
ledger), run **once** on that environment before the first
`db:migrate`:

```bash
bun run db:bootstrap
```

This inserts ledger rows so Drizzle treats the baseline migration as
already applied. Idempotent — safe to re-run.

## Why we don't use `db:push` anymore

`drizzle-kit push` skips the migration file step, so changes are not
version-controlled and cannot be reviewed in PRs or replayed
deterministically. It's intentionally not exposed as an npm script.
If you really need it for a throwaway sandbox, run `bunx drizzle-kit
push` directly — but never against a database that other people use.
